# Roadmap

## Near term

- Keep the shortcut set small.
- Improve README and examples after real use.
- Add more project examples only when needed.
- Keep the existing AutoHotkey / `install.ps1` usage compatible while the repository grows.
- Prepare GitHub automation docs, templates, and directory boundaries before moving implementation code.

## Later

- Add versioned releases if the kit is used on multiple PCs.
- Consider a one-command installer.
- Consider separating user-specific aliases from shared templates.
- Add shared GitHub Action or reusable workflow implementations one feature at a time.
- Add config schema validation and tests for auto-review / auto-merge workflows.
- Migrate proven consumer repository automation only through follow-up Issues and PRs.
