# GitHub上の文章言語

このドキュメントは、ChatGPTやCodexが作成するGitHub上の人向け文章について、別チャットや新しいタスクでも継続して適用する共通ルールです。

## 基本ルール

GitHub上で人が読む文章は、特別な理由がない限り日本語で作成します。

対象:

- Issueのタイトル・本文・コメント
- PRのタイトル・本文
- PRレビュー本文・レビューコメント
- Codexへの依頼文・修正指示・作業報告
- 作業状況コメントと引き継ぎ文
- Validation / Not run / Safetyなどの説明文
- GitHub Actionsが生成する人向けコメントやsummary

## 英語のままでよいもの

次は技術的な正確性や機械連携を優先し、英語のままで構いません。

- コード、識別子、変数名、関数名
- ファイルパス、ブランチ名、コマンド
- API名、HTTP status、ライブラリ名、製品名
- エラーコード、外部仕様で固定された文言
- GitHub markerや機械判定用metadata

技術用語を英語で残す場合も、周囲の説明文は日本語にします。

## Codexの作業開始時

Codexは新規タスクの開始時に`AGENTS.md`と本ドキュメントを確認し、以下を守ります。

- IssueとPRのタイトル・本文を原則日本語で作成する
- PR本文の人向け見出しも原則日本語にする
- レビュー対応、修正報告、Codex依頼文を日本語で作成する
- 英語で作成済みのIssueやPRを更新する場合、機械連携を壊さない範囲で日本語へ直す
- 言語統一だけを理由にコード、marker、metadata、API契約を変更しない

## 例外

外部ツール、GitHub Actions、API、既存parserなどが固定見出しや固定文字列を必要とする場合は、その文字列を維持します。その場合は、固定文字列以外の説明を日本語にし、必要なら理由をPR本文へ記録します。

## 正本

チャット内の記憶ではなく、`AGENTS.md`と本ドキュメントを正とします。別チャットや新しいCodexタスクでも、このリポジトリを扱う際は同じルールを適用します。
